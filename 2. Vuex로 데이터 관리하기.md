# 1. Vuex Store

- https://ko.nuxtjs.org/guide/vuex-store/
- 일반적으로는 클래식 모드를 사용하지만,
- 여기서는 파일 분리가 쉽고, 관리가 쉬운 모듈모드를 사용한다.

## 1. store 폴더 생성

- index.js, post.js, user.js 파일 생성
  - index는 기본 파일 이름이다.
- vuex에서 store폴더가 생성됨을 보고 알아서 빌드해준다.

```js
// front/store/posts.js

export const state = () => ({
  name: 'vuex',
})

export const mutations = () => ({
  bye(state) {
    state.name = 'goodbye';
  }
})
```

- 예시를 위해 default.vue에 아래의 입력한뒤, 홈페이지에서 확인한다. (name이 변경된다.)

```js
// front/store/posts.js

export const state = () => ({
  name: 'posts',
});

export const mutations = {
  bye(state) {
    state.name = 'goodbye posts';
  }
};
```

```html
// front/store/default.vue

<template>
  <div>
      {{ name }}
      <v-btn @click="bye">다음에 봐요</v-btn>
  </div>
</template>

<script>
  export default {
    computed: {
      name() {
        return this.$store.state.posts.name;
      },
    },
    methods: {
      bye() {
        this.$store.commit('posts/bye')
      }
    },
  }
</script>
```
